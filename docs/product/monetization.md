# Монетизация — ипотечный разбор

Основная монетизационная модель хакатона: **единый продукт «Ипотечный разбор»**, не набор модулей.

## Что делает пользователь

Не выбирает модули. Говорит, например:

- «Хочу ипотеку 12 млн, доход такой-то, подушка такая-то»
- или проще: **«Хочу понять, потяну ли ипотеку»**

Данные подтягиваются из финансовой модели (онбординг + поведение) или уточняются в диалоге.

## Что делает система (один ответ)

### 3.1 Шанс одобрения

Высокий / средний / низкий + почему (доход, нагрузка, подушка, стабильность).

### 3.2 Безопасный предел

- Максимальная сумма ипотеки
- Комфортный ежемесячный платёж

### 3.3 Риск нагрузки

- Ухудшение финансовой устойчивости
- Сколько месяцев подушки останется
- Насколько ситуация станет «хрупкой»

### 3.4 Сценарий «если взять сейчас»

Платежи, влияние на цели, риск перегруза.

### 3.5 Сценарий «если подождать»

Через сколько месяцев безопаснее, рост шанса одобрения, улучшение здоровья.

### 3.6 Сравнение банков (ключевое для демо)

3–5 банков: платежи, переплата, итоговая стоимость.

### 3.7 Оптимальный вариант

Лучший банк и баланс риск / платёж / переплата.
