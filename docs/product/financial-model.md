# Единая финансовая модель

Формируется **постепенно** из онбординга и каждого нового действия.

## Структура модели

| Компонент | Источник |
|-----------|----------|
| Активный доход | Онбординг, профиль |
| Пассивный доход | Онбординг, профиль |
| Фактические расходы | Голос, чеки, ФНС, LK |
| Обязательные расходы | Онбординг (опционально) |
| Подушка безопасности | Онбординг, обновления |
| Цель | Онбординг, goal-service |

## Расчёты (логика продукта)

- **Свободный денежный поток** = доход − расходы
- **Runway** = подушка / среднемесячные расходы
- **Скорость цели** = f(свободный поток, цель, текущие накопления, поведенческие паттерны)
- **Влияние траты** = сдвиг срока цели на N дней/месяцев

## Основной цикл

1. Пользователь совершает действие (добавил расход, изменил цель, подключил ФНС)
2. Система интерпретирует (`ai-processor`, `receipt-service`)
3. Обновляет модель (PostgreSQL + агрегаты ClickHouse)
4. Показывает влияние на цель (`analytics-service`, dashboard)
5. Предлагает **одно** мягкое улучшение

## Микро-действия

Одно предложение после анализа:

> Если уменьшить доставку еды, цель станет ближе на 10 дней.

## Хранение (`back`)

| Данные | Таблица / сервис |
|--------|------------------|
| Пользователь, провайдеры | `users`, `user_providers` |
| Чеки, позиции | `receipts` (JSONB items) |
| Ручной/голос | `manual_expenses` |
| Цели | `goals` |
| Кредиты, DTI | `credits` |
| Агрегаты | ClickHouse `receipt_items`, MV |
