# Онбординг (~1 минута)

## Цель

Не собрать данные идеально, а построить **рабочую финансовую модель** и показать первый осмысленный результат.

## Маршрут на front

| Элемент | Значение |
|---------|----------|
| **URL** | `/onboarding` |
| **Когда** | Первый вход после регистрации / login (профиль не заполнен) |
| **После завершения** | Редирект на `/dashboard` |
| **Middleware** | Если `onboarding_completed === false` → только `/`, `/login`, `/onboarding` |

> **Статус:** страница **⏳ к реализации** на ветке `front`. Спека ниже — источник правды для front-команды.  
> См. также [NAVI.md](../../NAVI.md) → раздел «Фронтенд».

### Структура страницы (wizard, один экран — один шаг)

```
/onboarding
  ├── step 1: welcome + «Начать»
  ├── step 2: доходы (активный + пассивный)
  ├── step 3: подушка (сумма + опциональная разбивка)
  ├── step 4: цель (тип + сумма)
  ├── step 5: обязательные расходы (опционально, «Пропустить»)
  ├── step 6: первая картина + первое действие
  └── step 7: первый ввод (ФНС / вручную / голос) — одно действие
```

Layout: отдельный от app-shell (без sidebar), спокойный фон как на welcome.

## Шаги (контент)

### Старт

> Поток помогает понять, как твои деньги работают на твои цели.

Кнопка **«Начать»**.

### Доходы

- Активный доход (в месяц)
- Пассивный доход (в месяц)

### Подушка безопасности

- Общая сумма
- Разбивка (опционально): наличные, вклад, инвестиции

### Цель

- Тип: накопить сумму / покупка / подушка / другое
- Сумма цели

### Регулярные расходы (опционально)

Только крупные и фиксированные: аренда, парковка, кредиты, обязательные платежи.  
Формат: название + сумма в месяц.

**Если пропустил:**

> Мы учтём твои расходы по поведению.

### Финал онбординга

Система показывает **текущую финансовую картину** и **примерный путь к цели** — не аналитику как сухой факт, а **первое действие**.

## Первое взаимодействие (одно простое действие)

Пользователь **не обязан** подключать интеграции. Варианты:

| # | Действие | UI |
|---|----------|-----|
| 1 | Подключить ФНС | «Подтянуть автоматически» |
| 2 | Внести последнюю покупку | «Добавить вручную» |
| 3 | Надиктовать голосом | «Сказать голосом» |

После ввода: разбивка → категоризация → модель → редирект `/dashboard`.

## Связь с back

| Шаг | API / сервис |
|-----|----------------|
| Профиль, доходы | `user-service` (расширить профиль или `PATCH /api/v1/users/me`) |
| Цель | `POST /api/v1/goals/` → goal-service |
| Обязательные расходы | budgets или поле профиля (roadmap) |
| Ручной / голос | `POST /api/v1/expenses/manual` → ai-processor |
| ФНС | `POST /api/v1/fns/*` → scraper-service |
